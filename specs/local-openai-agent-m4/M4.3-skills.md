# M4.3: Local Agent Skill Loading

## Summary

Add feature-flagged local skill loading for the local OpenAI-compatible agent. The agent discovers Claude-style local skill packages, injects bounded skill metadata into the system prompt, and loads full `SKILL.md` content only when the user explicitly invokes a skill such as `/skill-name`.

Current branch note: Warp already has skill parsing and management primitives under `crates/ai/src/skills/*` and `app/src/ai/skills/*`, plus slash-command skill invocation paths. M4.3 should reuse those primitives while constraining local-agent behavior to local filesystem skills.

## User Story

As a Warp user, I want my local OpenAI-compatible agent to use project and personal skill instructions, so I can keep reusable workflows near my code or home configuration without sending them through a hosted agent service.

## Acceptance Criteria

1. Add `FeatureFlag::LocalAgentSkills`, default off, following the existing local-agent feature flag pattern.
2. When `LocalAgentSkills` is off, local OpenAI-compatible system prompts, tool definitions, and slash behavior are unchanged.
3. When the flag is on, discover local skills from:
   - `~/.claude/skills/<name>/SKILL.md`
   - `<project>/.claude/skills/<name>/SKILL.md`
4. Project-level skills override user-level skills with the same normalized `name`.
5. User-level duplicate names resolve deterministically by discovery order and emit a warning log that does not include full skill content.
6. `SKILL.md` frontmatter validation for local-agent injection:
   - `name` is required and must be kebab-case.
   - `description` is required and must be 200 characters or fewer.
   - Optional `metadata` may be parsed but must not affect execution privileges in M4.3.
7. Invalid skill files are skipped with bounded diagnostics. A bad skill must not fail the whole local-agent request.
8. Inject only skill metadata into the default local system prompt:
   - Skill name.
   - Description.
   - Source scope, such as project or user, if useful for disambiguation.
   - Do not inject full `SKILL.md` contents for every request.
9. Skill metadata injection must be bounded:
   - Enforce a maximum number of injected skills.
   - Enforce per-skill and total prompt length limits.
   - Prefer project skills over user skills when truncation is necessary.
10. A user input that explicitly invokes `/skill-name` loads the full corresponding `SKILL.md` content into the local request context for that turn.
11. Explicit skill invocation must preserve the user's remaining query text and route through the local OpenAI-compatible provider when the selected agent is local.
12. Full skill content is prompt-as-data:
   - It may instruct the model.
   - It must not automatically execute shell commands, write files, call MCP tools, or load resources without normal tool policy and user approval.
13. Skill resources are lazy-loaded only when explicitly referenced by a supported local-agent path. Resource paths must canonicalize under the skill package directory and reject symlink/path traversal escapes.
14. M4.3 does not add remote marketplace install, update, fetch, or trust-cache behavior.
15. M4.3 does not add executable plugin hooks. Files such as scripts may be read as resources only when explicitly allowed and path-contained; they must not be run automatically.
16. If a slash skill name conflicts with a built-in slash command, the built-in command wins unless existing Warp skill invocation semantics already define a safer explicit skill route.
17. Skill content, local filesystem paths outside the skill root, and resource contents must not be written to product telemetry or unsafe logs.
18. Existing local-agent tests continue to pass. Add focused tests for:
   - Feature flag off omits skill metadata.
   - User and project skill discovery.
   - Project override of user skill.
   - Frontmatter validation for kebab-case name and description length.
   - Bounded metadata injection.
   - `/skill-name` loads full `SKILL.md` content for that turn only.
   - Resource path traversal and symlink escape fail closed.
   - Invalid skills are skipped without failing the request.

## Safety Risks

- Prompt injection from local files: skill content can contain arbitrary instructions. Mitigate by treating skills as user-provided prompt data and preserving existing tool approval/sandbox rules.
- Automatic execution: skill packages may include scripts. Mitigate by never running skill resources automatically.
- Path traversal: resources could reference files outside the skill package. Mitigate with canonicalization and root containment checks.
- Prompt bloat: loading every full skill would consume context and leak unnecessary local instructions. Mitigate with metadata-only default injection and explicit full-content loading.
- Command conflict: slash skill names could shadow built-in commands. Mitigate by preserving built-in slash command precedence.

## Scale Estimate

M/L. Skill parsing and management already exist, but M4.3 needs local-agent-specific discovery constraints, bounded prompt injection, slash-triggered full content loading, path safety, and tests.

## PR Split Suggestion

Keep M4.3 as one implementation PR with at most seven commits:

1. Add `LocalAgentSkills` feature flag, default off.
2. Add local-agent skill discovery adapter using existing parser/manager primitives and Claude-style paths.
3. Add frontmatter validation and deterministic conflict handling for local-agent injection.
4. Add bounded metadata injection into local system prompt.
5. Wire `/skill-name` invocation to load full `SKILL.md` content for local provider requests.
6. Add safe lazy resource resolution with root containment checks if resource loading is included in M4.3.
7. Add tests for discovery, validation, injection, invocation, conflicts, and path defense.

Each commit should compile independently. If resource loading expands the diff too much, keep M4.3 to `SKILL.md` loading and write a follow-up spec for resources.

## Entry Points

- `crates/ai/src/skills/parse_skill.rs`: existing `SKILL.md` parser and frontmatter behavior.
- `crates/ai/src/skills/read_skills.rs`: filesystem reading helpers.
- `crates/ai/src/skills/skill_provider.rs`: provider abstractions and descriptor types.
- `app/src/ai/skills/skill_manager.rs`: cached skill discovery, scoping, and deduplication.
- `app/src/ai/skills/resolve_skill_spec.rs`: existing skill reference resolution.
- `app/src/ai/blocklist/controller/slash_command.rs`: slash skill invocation and command precedence.
- `app/src/ai/blocklist/action_model/execute/read_skill.rs`: existing full skill read action semantics.
- `app/src/ai/agent/local/mod.rs`: local system prompt construction and request context injection.
- `app/src/ai/agent/api.rs`: request context shape if skill content needs to travel through local params.
- `app/Cargo.toml`, `app/src/lib.rs`, `crates/warp_features/src/lib.rs`: feature flag plumbing.
