# M3.2: Local Agent Token and Cost Telemetry

## Summary

Capture token usage from local OpenAI-compatible provider responses, estimate request cost locally, and surface that usage through Warp's existing conversation usage footer. This is user-visible local accounting only; it must not upload local usage or pricing data to Warp telemetry.

Current branch note: `stream_finished_done()` in `app/src/ai/agent/local/mod.rs` emits empty token usage, `request_cost: None`, and zeroed `conversation_usage_metadata`.

## User Story

As a Warp user using a local OpenAI-compatible provider, I want to see token usage and estimated cost for each local agent response, so I can understand model spend and diagnose unusually expensive prompts without sending that accounting data to Warp servers.

## Acceptance Criteria

1. Add `FeatureFlag::LocalAgentCostTelemetry`, default off, following the M1/M2 feature flag pattern.
2. When the flag is off, local OpenAI-compatible streams continue to emit the existing zero/empty usage metadata shape.
3. When the flag is on, local direct requests ask for streaming usage where supported:
   - Add `stream_options: { "include_usage": true }` to OpenAI-compatible chat completion requests.
   - Parse any `usage` object present on stream chunks even if the provider sends it without `stream_options`.
4. If a provider rejects `stream_options` as an unknown or unsupported parameter before streaming begins, retry the request once without `stream_options` and mark usage as unavailable for that request. Do not retry after any response tokens have been streamed.
5. Parse at least these usage fields:
   - `prompt_tokens`
   - `completion_tokens`
   - `total_tokens`
   - Optional cache fields when providers expose OpenAI-style prompt token details.
6. Missing, null, malformed, or partial usage must not fail the user request. Finish normally and surface "usage unavailable" or omit the usage footer for that request.
7. Convert provider usage into Warp's existing `StreamFinished` fields:
   - `token_usage` for backward-compatible per-model totals.
   - `conversation_usage_metadata.byok_token_usage` because local OpenAI-compatible uses a user-provided key/base URL.
   - `PRIMARY_AGENT_CATEGORY` for the local agent model usage category.
8. `request_cost` and `conversation_usage_metadata.credits_spent` represent a local estimate, not Warp credits:
   - Use estimated USD cents internally for calculation.
   - Display should clearly say "estimated" or use existing BYOK wording if available.
   - Do not decrement Warp request credits or refresh `AIRequestUsageModel` based on local usage.
9. Pricing source recommendation:
   - Ship a small built-in pricing registry keyed by provider family and model id/pattern for common OpenAI-compatible models.
   - Add a local user override path in settings/preferences for exact input/output/cache prices per million tokens.
   - Missing price means token counts still display and cost is unknown/omitted.
   - Do not fetch live pricing from third-party providers in M3.2.
10. Pricing registry entries must include an `effective_date` or comment/date in code so future updates are auditable. Unit tests must not assert market prices beyond deterministic fixture values.
11. Cost calculation must account for:
   - Input/prompt tokens.
   - Output/completion tokens.
   - Cache read/write tokens if both usage and pricing are available.
   - Unknown fields ignored safely.
12. The usage footer should reuse existing `ConversationUsageView` and `AIConversation::update_cost_and_usage_for_request` paths where practical. Avoid adding a parallel local-only usage panel unless existing structures cannot represent estimated BYOK usage.
13. Local usage/cost must persist with the conversation using existing conversation usage metadata persistence when available.
14. Privacy constraints:
   - Do not log raw prompts, completions, image bytes, tool results, or provider usage JSON.
   - Do not send local usage/cost estimates to product telemetry, Warp billing APIs, or analytics.
   - Safe logs may include model id and aggregate token counts only when debug logging already permits similar metadata.
15. Existing M2 local-agent tests continue to pass. Add focused tests for:
   - Request JSON includes `stream_options.include_usage` only when flag is on.
   - Usage is parsed from final stream chunk.
   - Missing/malformed usage finishes normally.
   - Unsupported `stream_options` retries once without duplicating streamed output.
   - Token usage maps to BYOK `PRIMARY_AGENT_CATEGORY`.
   - Cost estimate handles known price, missing price, and cache tokens.
   - `stream_finished_done` emits populated metadata when usage is available.

## Safety Risks

- Privacy regression: token telemetry could become product telemetry. Mitigate by keeping all M3.2 accounting local and avoiding telemetry event emission.
- Misleading cost: provider prices and token accounting vary. Mitigate with "estimated" copy, user overrides, effective dates, and unknown-cost fallback.
- Double request: retrying after stream start could double-charge or duplicate effects. Mitigate by retrying only before any chunk content is processed and only for unsupported `stream_options`.
- Provider incompatibility: OpenAI-compatible providers differ in usage field names. Mitigate with tolerant parsing and no-fail missing usage.
- UI confusion with Warp credits: local BYOK cost is not Warp credits. Mitigate by preserving BYOK category and avoiding request credit updates.

## Scale Estimate

M/L. The data extraction is contained in local streaming, but the work crosses request serialization, SSE parsing, pricing calculation, `StreamFinished` metadata, conversation persistence, and usage footer display semantics.

## PR Split Suggestion

Keep M3.2 as one PR if the UI can reuse existing usage footer semantics; otherwise split the display-only change into a follow-up. Target at most seven commits:

1. Add `LocalAgentCostTelemetry` feature flag, default off.
2. Add OpenAI-compatible usage structs and tolerant stream parsing.
3. Add `stream_options.include_usage` request support and one-shot unsupported-parameter fallback.
4. Add local pricing registry and user override model with deterministic tests.
5. Map usage and cost into `StreamFinished` token/cost metadata.
6. Reuse/update usage footer display copy for estimated local BYOK usage.
7. Add regression tests for missing usage, malformed usage, cache fields, and no product telemetry emission.

Each commit should be independently testable and should not alter remote/Oz server usage accounting.

## Entry Points

- `app/src/ai/agent/local/mod.rs`: `OpenAIChatRequest`, `OpenAIStreamResponse`, SSE parsing, request retry, usage accumulation, and `stream_finished_done`.
- `app/src/ai/agent/conversation.rs`: existing `update_cost_and_usage_for_request` behavior for local `StreamFinished` metadata.
- `app/src/ai/blocklist/usage/conversation_usage_view.rs`: display wording for BYOK/local estimated usage if current copy is ambiguous.
- `crates/persistence/src/model.rs`: `ModelTokenUsage`, token usage categories, and persisted conversation usage metadata.
- `app/src/pricing/mod.rs` or a new local pricing module under `app/src/ai/agent/local/`: local model price registry and user override plumbing.
- `ai::api_keys::ApiKeyManager` / settings models: local override storage if implementation chooses settings-backed prices.
- `app/Cargo.toml`, `app/src/lib.rs`, `crates/warp_features/src/lib.rs`: feature flag plumbing.
